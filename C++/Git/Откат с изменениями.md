Более мягкие ключи `--soft` и `--mixed` тоже позволяют команде `git reset` откатить репозиторий к выбранному коммиту. При этом файлы репозитория остаются в неизменном виде. Отличие будет заключаться в их состоянии.

Исследуйте ключ `--soft`.

```sh
$ git reset --soft ab17582

$ git log --oneline
ab17582 (HEAD -> master) Добавлена закрывающая скобка
287abde first commit

$ git status
On branch master
Your branch is behind 'origin/master' by 3 commits, and can be fast-forwarded.
  (use "git pull" to update your local branch)

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        new file:   .gitignore 
```

Коммитов `0c7a6c0..d9b4816` не осталось в репозитории. При этом файл .gitignore отмечен как новый и уже добавлен в индекс. Остаётся только сделать коммит, чтобы зафиксировать изменения.

Последний ключ — `--mixed`. На самом деле этот ключ применяется по умолчанию. То есть команды `git reset` и `git reset --mixed` равносильны.

```sh
$ git reset --mixed ab17582

$ git log --oneline
ab17582 (HEAD -> master) Добавлена закрывающая скобка
287abde first commit

$ git status
On branch master
Your branch is behind 'origin/master' by 3 commits, and can be fast-forwarded.
  (use "git pull" to update your local branch)

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        .gitignore

nothing added to commit but untracked files present (use "git add" to track) 
```

В индекс не добавлено никаких изменений, поэтому Git подсказывает, что надо использовать команду `git add`.

Команда `git reset` с ключами `--soft` и `--mixed` позволяет более аккуратно выполнять откаты. Все изменения остаются в файлах. Остаётся только оставить эти изменения или удалить их.
