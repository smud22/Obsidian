Некоторых файлов и директорий не должно быть даже в списке неотслеживаемых. Например, файлов с вашими паролями или настройками конкретного компьютера. Если репозиторий публичный, то злоумышленники могут воспользоваться вашими секретными данными и взломать личные аккаунты или рабочие проекты. Чтобы эти файлы не попали в удалённый репозиторий, используйте текстовый файл .gitignore.

Достаточно указать в файле .gitignore название директории или имя файла, и Git перестанет их видеть. Если игнорируемый файл находится не в корне проекта, нужно указать путь до него.

```sh
# Игнорировать файл README.md.
README.md

# Игнорировать файл side.txt в директории build.
build/side.txt

# Игнорировать все файлы с расширением .doc.
*.doc 
```

В директории проекта cpp-test-repo создайте файлы text.doc и .gitignore. Для создания файлов можно воспользоваться командой `touch`:

```sh
touch .gitignore 
```

В файл .gitignore добавьте указание `*.doc`.

Зайдите в терминал, перейдите в директорию cpp-test-repo и выполните два действия:

1. Добавьте изменения в индекс — `git add .`.
2. Сделайте коммит с сообщением «Добавлен файл .gitignore».

В каталоге появилось два новых файла, но отслеживается только один.
