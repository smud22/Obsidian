Чтобы получить информацию о коммитах проекта cpp-test-repo, находясь в любой папке проекта, выполните команду `git log` (англ. log — «журнал»).

```sh
# Команда для просмотра истории коммитов.
git log

# Идентификатор коммита.
commit d9b48163447db6da8f31447693469f84a2cb1503 (HEAD -> master, origin/master, origin/HEAD)
# Автор коммита.
Author: User <user@yandex.ru>
# Дата проведения коммита.
Date:   Mon Oct 23 10:46:17 2023 +0300

    # Сообщение к коммиту.
    Удалён файл bad.cpp

commit 8e718b966523183b8460c27549572a580905219b
Author: User <user@yandex.ru>
Date:   Mon Oct 23 10:44:19 2023 +0300

    Добавлен файл bad.cpp

commit 0c7a6c0e41cc7a2bb5ba247e2c7c4f28d6d6f9cc
Author: User <user@yandex.ru>
Date:   Mon Oct 23 10:38:38 2023 +0300

    Добавлен файл .gitignore

commit ab17582464548e807d38996d37aee28914377321
Author: User <user@yandex.ru>
Date:   Mon Oct 23 10:26:48 2023 +0300

    Добавлена закрывающая скобка 
```

Когда информации будет много, чтобы не пролистывать все строки до конца, для выхода из режима просмотра журнала можно нажать клавишу Q (англ. quit — «выйти»).

У команды `git log` есть несколько особенно полезных ключей:

- `-n <number>` — вывести определённое количество коммитов;
- `--author=<pattern>` — вывести коммиты определённого автора;
- `--oneline` — вывести информацию о коммитах в сокращённом формате.

```sh
# Вывести информацию о 3 коммитах.
git log -n 3

# Вывести информацию о коммитах авторов с почтой на домене yandex.ru.
git log --author="@yandex.ru"

# Получить информацию о коммитах в кратком виде.
git log --oneline

# Ключи можно объединять.
# Вывести информацию о 3 коммитах автора User в кратком виде.
git log -n 3 --author="User" --oneline 
```
