Git — это машина времени. Git может вернуть ваш код в любое предыдущее состояние, если оно сохранено в коммите. Для этого есть команда `git reset` (англ. reset — «сброс в исходное состояние»). Но не спешите её применять. Выполнение команды с разными ключами ведёт за собой разные последствия. Начнём с самого «тяжёлого» ключа `--hard`.

Чтобы вернуться к определённому коммиту, нужно выполнить команду `git reset --hard` и через пробел указать первые семь символов идентификатора нужного коммита.

```sh
# Откатить до коммита, в котором добавлена закрывающая скобка.
git reset --hard ab17582 
```

Эта команда удалит все последующие коммиты и перезапишет файлы репозитория.

Команда `git reset --hard` считается опасной, потому что полностью меняет состояние файлов и может привести к утере полезных изменений. Команда является необратимой, если у вас нет отдельной копии или удалённого репозитория.

Посмотрите, что стало с историей коммитов.

```sh
$ git log --oneline

ab17582 (HEAD -> master) Добавлена закрывающая скобка
287abde first commit 
```

Коммит `ab17582` теперь последний. А в каком состоянии файлы репозитория?

```sh
$ git status

On branch master
Your branch is behind 'origin/master' by 3 commits, and can be fast-forwarded.
  (use "git pull" to update your local branch)

nothing to commit, working tree clean 
```

Локальный репозиторий отличается от удалённого на три коммита, поэтому Git предлагает обновить локальную ветку. Но это сейчас не имеет значения. Важно, что коммитить нечего, никаких изменений нет.
