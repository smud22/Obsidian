Команда `git restore` применяется и без дополнительных ключей. Её можно использовать, чтобы вернуть файл в последнее зафиксированное состояние. То есть состояние после применения команды `git add` или `git commit`.

```sh
# Поменяли файл main.cpp.
$ git status
On branch master
Your branch is up to date with 'origin/master'.

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   main.cpp

no changes added to commit (use "git add" and/or "git commit -a")

$ git restore main.cpp
$ git status
On branch master
Your branch is up to date with 'origin/master'.

nothing to commit, working tree clean 
```

Всегда обращайте внимание на подсказки, которые предлагает Git при вызове команды `git status`. Они напомнят о возможных командах в текущий момент и синтаксисе этих команд.
