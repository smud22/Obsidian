До этого момента слот на выбранный сигнал вы устанавливали через дизайнер. Действовали так:

- Выбирали объект, который будет источником сигнала.
- Выбирали сигнал, который будет выполняться. Например, `clicked()` для реакции на нажатие кнопки.

Затем дизайнер создавал слот в вашем классе, который выполнял какие-либо действия.

В этом процессе участвуют два объекта (источник сигнала — например, кнопка и получатель сигнала — класс окна), слот и сигнал. Получается такая схема:

![](https://code.s3.yandex.net/CPP/assets/c01_2024/s02_qt_basics/connect.png)

_Схема взаимодействия сигнала и слота_

Но `QTimer` не виджет, по которому вы бы могли щёлкнуть в дизайнере. Поэтому будем использовать программный способ связывания сигнала и слота, для которого понадобится метод `connect`. Метод `connect` принимает на вход четыре параметра — столько же, сколько блоков на схеме:

- **Объект** — источник сигнала. Для таймера из примера первым параметром будет `&timer_for_one_sec`. Амперсанд (`&`) ставится, так как это объект, созданный нами в классе окна.
- **Сигнал**, в нашем случае это сигнал `timeout` класса `QTimer`. Записывается это так: `&QTimer::timeout`. В этом случае амперсанд обозначает ссылку на функцию.
- **Объект-приёмник**, в нашем случае – объект класса `MainWindow`. Укажем `this`, поскольку пишем код внутри метода этого класса.
- **Вызываемый слот**. Нужный слот добавим в класс `MainWindow`. Мы решили назвать его `on_timer_timeout`. Чтобы сослаться на добавленный слот напишем так: `&MainWindow::on_timer_timeout`.


В итоге получится такая строка кода:

```cpp
// Подключаем сигнал таймера к слоту on_timer_timeout.
connect(&timer_for_one_sec, &QTimer::timeout, this, &MainWindow::on_timer_timeout); 
```

Если вы разместите такой код в своём проекте и попробуете откомпилировать его, то получите ошибку компиляции:

![](https://code.s3.yandex.net/CPP/assets/c01_2024/s02_qt_basics/time_1.png)

Эта ошибка говорит, что в классе не определён слот с названием `on_timer_timeout`. Слоты принято добавлять в область со специальным названием `private slots:`. Добавьте слот в класс. После этого проект должен откомпилироваться успешно.

```cpp
class MainWindow : public QMainWindow
{
    Q_OBJECT

public:
    MainWindow(QWidget *parent = nullptr);
    ~MainWindow();

private slots:
  // Добавление слота в класс, который обработает принятый сигнал.
  void on_timer_timeout() {
      // Код, выполняющий какие-либо действия.
  }
  ....
}; 
```
