Qt реагирует на действия пользователя двумя способами. Один вам уже знаком — сигналы и слоты. Другой — события. Например, существует событие `resizeEvent` — изменение размеров окна. Чтобы реагировать на это событие, нужно определить в классе окна метод `void resizeEvent(QResizeEvent* event)`. Он определяется как обычный метод, но при декларации добавляется слово `override`:

```cpp
class MainWindow : public QMainWindow
{
    // ...
    
private:
    void resizeEvent(QResizeEvent* event) override;

    // ...
}; 
```

Класс `MainWindow` имеет базовый класс `QMainWindow`. Это значит, что в нём доступно всё, что есть в `QMainWindow`, в том числе метод `resizeEvent`. Слово `override` показывает, что мы заменяем метод базового класса своим методом. Оно нужно, чтобы компилятор проверил, что мы нигде не ошиблись, и метод базового класса действительно заменился. Более подробно про базовые классы и переопределение методов вы узнаете из темы про объектно-ориентированное программирование.

Как только пользователь изменит размер формы, вызовется метод `resizeEvent`. В единственный параметр `event` будет передан указатель на объект события, содержащий всю информацию о том, что произошло. При этом можно выполнить какие-то действия, например, адаптировать элементы окна под новый размер.

Кроме события `resizeEvent` есть и другие. Например, событие `keyPressEvent` (см. [документацию](https://doc.qt.io/qt-6/qwidget.html#keyPressEvent)) произойдёт при нажатии кнопки на клавиатуре. Его можно использовать, чтобы добавить в программу поддержку горячих клавиш. Информация о том, какую именно клавишу нажал пользователь, будет в параметре типа `QKeyPressEvent`.

Напишем программу, которая покажет размер окна в реальном времени. Для этого добавим метод `UpdateSizeLabel`, он обновит текст надписи:

```cpp
void MainWindow::UpdateSizeLabel() {
    auto new_text = std::to_string(width()) + "x" + std::to_string(height());
    ui->lbl_size->setText(QString::fromStdString(new_text));
} 
```
