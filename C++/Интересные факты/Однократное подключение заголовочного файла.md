Практически всегда многократное подключение заголовочного файла в одной единице трансляции бесполезно, а в ряде случаев может привести к ошибкам компиляции. Чтобы защититься от этого, в начале заголовочного файла пишут директиву `#pragma once`. Она поддерживается всеми современными компиляторами и означает: не подключай этот файл повторно.

```cpp
// Файл factorial.h

// Просто пишите эту директиву в каждом заголовочном файле.
#pragma once

int Factorial(int n); 
```

Теперь, сколько бы раз вы ни подключали factorial.h, последующие включения будут проигнорированы.

```cpp
// Файл main.cpp

#include "factorial.h"
// Из-за #pragma once в factorial.h эта директива #include будет проигнорирована.
#include "factorial.h"

#include <iostream>
// Повторное включение стандартных заголовочных файлов также игнорируется.
#include <iostream> 

int main() {
    std::cout << Factorial(3) << std::endl;
} 
```

Если у параметров функции есть значения по умолчанию, как у функции `StringToNumber`, то указывать их нужно только в заголовочном файле. В файле с определением функций они указываться не должны. Иначе будет ошибка компиляции.

```cpp
// Файл notation.cpp

#include "notation.h"

/* Здесь для параметра base не указано значение по умолчанию. */
int StringToNumber(const std::string& s, bool& was_error, int base) {
    ...
}

/* Прочие функции. */ 
```
