Изменяемые значения называют **l-value** (от англ. left, «левый»), потому что они могут стоять слева от знака равенства. Неизменяемые называют **r-value** (от англ. right, «правый»), потому что они могут стоять только справа от знака равенства. Например, в выражении `x = 10` значение `x` — изменяемое, его можно изменить, а 10 — неизменяемо. Нельзя поменять число 10.

Рассмотрим пример:

```
int a = 4;
int b = ++(++a); // a и b равны шести. 
```

Теперь рассмотрим другой пример:

```
int a = 4;
int b = (a++)++; // Ошибка компиляции! 
```

Постфиксная операция, находящаяся внутри скобок, возвращает значение `а`, равное четырём, и увеличивает `а` до пяти. Внешняя операция при этом получает «старое» значение переменной `а` в виде неизменяемого значения. Получается ошибка — нельзя увеличивать неизменяемое значение.

Можно неограниченно выполнять префиксные операции из других префиксных операций, но постфиксная операция не может быть вложенной ни в префиксную, ни в постфиксную.

Операции `++` и `--` могут приводить к неопределённому поведению. Например, подумайте, чему будет равно значение выражения `y + y++` при исходном `y = 0` . Если исходное значение переменной `y` будет подставлено сразу, то получится `0 + 0`, то есть 0. Иначе если компилятор решит, что сначала нужно вычислить `y++`, то получится `1 + 0`, то есть 1. Поэтому применять эти операции нужно внимательно.
