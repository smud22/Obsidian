Примитивные типы данных (`bool`, `char`, целые и вещественные числа) всегда выгоднее передавать по значению. Структуры размером не более 16–32 байт, содержащие примитивные поля, обычно также легко передаются по значению. Структуры большего размера, `std::string` и `std::vector` выгоднее передавать по константной ссылке, а не по значению.

Какие типы выгоднее передавать по константной ссылке в функцию, которая не модифицирует свой аргумент?

```cpp
struct RGBAColor {
    // Предполагается, что размер float равен 4 байтам.
    float red, green, blue, alpha;
}; 
``` 

```cpp
struct Point {
    // Предполагается, что размер double равен 8 байтам.
    double x, y;
}; 
``` 

Правильный ответ

```cpp
struct Student {
    std::string name, surname;
    int age;
    std::vector<int> scores;
}; 
``` 

Структура содержит `std::string` и `std::vector`. Эти типы лучше передавать по константной ссылке.

```cpp
struct Circle {
    Point center;
    double radius;
}; 
``` 

```cpp
struct Rectangle {
    Point left_top;
    double width, height;
}; 
``` 

Правильный ответ

```cpp
struct Matrix3x3 {
    double m00, m01, m02;
    double m10, m11, m12;
    double m20, m21, m22;
}; 
``` 

Размер структуры — 72 байта. Она достаточно тяжёлая, чтобы выгоднее было передавать её по константной ссылке, а не по значению.

Правильный ответ

```cpp
std::string; 
```
