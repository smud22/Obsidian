Когда мы писали `std::vector<int> numbers`, то создавали пустой вектор. При этом использовался конструктор по умолчанию.

Можно создать вектор, в котором сразу будут находиться какие-то объекты. Для этого предусмотрены другие конструкторы, отличные от «дефолтного».

```cpp
// Так будет создан вектор из семи целых чисел.
// Он будет заполнен «дефолтными» значениями — нулями.
std::vector<int> seven_zeros(7);

// Вектор, содержащий 10 целых чисел, каждое из которых равно 50.
std::vector<int> ten_fifties(10, 50);

// Копия вектора ten_fifties.
std::vector<int> another(ten_fifties);

// Другой способ создать копию.
std::vector<int> another2 = ten_fifties;

// Явно задаём элементы.
std::vector<int> holiday{15, 100, -42, 100500}; 
```

Как сказано ранее, тип указывается при объявлении переменной. В этом случае переменная `days_of_the_week` имеет тип `std::vector<std::string>`. Значит, сам `std::vector` — это ещё не тип. Типом он становится, только если подставить в него аргумент, в данном случае `std::string`. Более того, `std::vector<int>`, `std::vector<std::string>`, `std::vector<char>` — это три разных типа.

Если `std::vector` — не тип, то что же? Это **шаблон**, то есть специальная конструкция языка, которая позволяет создать тип на основе **шаблонных аргументов**. В примерах выше шаблонные аргументы — это `int`, `std::string`, `char`. Тип, получающийся из шаблона, в котором задали аргументы, называется **специализацией шаблона**. Одна такая специализация вам уже знакома — `std::string`. Это не самостоятельный тип, а специализация более общего шаблона `std::basic_string`.

С++ — это язык с большими возможностями шаблонов, которые часто называют шаблонной магией. Создание шаблонов называют **обобщённым программированием**, потому что шаблон — общий код, который должен работать с разными конкретными типами. Пока что вам заниматься шаблонной магией не требуется, она понадобится позже.
