При написании алгоритмов используют понятие, которое помогает разработчику убедиться, что алгоритм работает верно, — **инвариант**. Это некоторое утверждение, которое формулируется в комментарии. При этом соблюдают правило: в момент, когда программа достигает комментария, утверждение должно быть верно. Приведём примеры инвариантов:

```cpp
bool is_first_iteration = true;

std::string word;
while (std::cin >> word) {
    // Инвариант: is_first_iteration имеет значение true, 
    //            только если сейчас первая итерация.
    if (!is_first_iteration) {
        std::cout << ", ";    
    }

    is_first_iteration = false;
    std::cout << word;

    // Инвариант: is_first_iteration == false.
}

// После цикла можно написать инвариант на основе условия.
// Ведь цикл завершается, только когда условие ложно,
// а значит, на следующей после цикла строке оно точно ложно.
// Инвариант: !std::cin. 
```

Компилятор не учитывает инварианты. Программисты пишут их для себя и для других программистов. Это помогает написать верный алгоритм или найти ошибку.
